# MoMorph — リリースノート

MoMorph（Plugin・Web・MCP Server）の最新アップデート。

## 2026-06-11

**✨ 新機能・改善**

- **メンテナンス事前通知** — メンテナンス開始前に、Plugin・Web で開始時刻（ユーザーのタイムゾーン表示）を知らせるモーダルを表示します。
- **Web で AI 仕様生成** — Screen Spec と View All Specs のアクションメニューから AI 生成が可能になり、Plugin と挙動を統一しました。
- **Figma の Group レイヤー対応** — タイプが Group（グループ化された要素）のレイヤーを、通常のデザインアイテムと同様にリンク・編集・管理できるようになりました。
- **柔軟な仕様入力** — 必須項目の制限を撤廃（必須は名前または No. のみ）し、多様なプロジェクトで自由に仕様を入力できるようになりました。
- 多言語ラベルの統一と、アイテムツールバーへの新ツールチップを Plugin・Web に追加。

**🐛 修正**

- 本番環境での AI 仕様生成を安定化。「上書きしない」設定で `item type` が上書きされる問題、`Name` 空欄がスキップされる問題、編集→リロード後の「items がフレームに存在しない」エラーを修正。
- スクリーン翻訳で原文のまま残っていた長文テキストも翻訳されるよう修正。
- UI パートの再リンクを修正 — アイテム間のレイヤー再割り当て、およびデザイナーがレイヤーを replace した後の再リンク。
- screen を新しいデザインへリンクし直す際のエラーを修正。
- アイテムの完了ステータスが正しく保持されるよう修正 —「入力を完了する」後に「作成完了」になり、「作成完了」から「生成完了」へ勝手に戻らなくなりました。
- その他：AI エラーアイテムの Save/Cancel ボタン、日本語・ベトナム語入力時の二重表記、Web Screen List の合計件数未表示、メンテナンス取得失敗トーストのユーザー表示を修正。また、Web でも missing アイテムの AI 仕様生成をブロック（Plugin と統一）。

**🔌 MCP サーバー**

- **破壊的変更 — レガシー `status` フィールドを削除**（`create_frame` / `get_frame` / `list_frames`）。今後は 4 つの `*_status` フィールド（`design_status` / `spec_status` / `dev_status` / `review_status`）を使用してください。`status` を読み書きする agent・workflow は更新が必要です。
- **新ツール `update_frame`** — フレームを作り直さずに、画面メタデータ（name・overview・各 status・Figma ノードリンク）を個別に更新できます。
- **`figma_node_id` 対応** — `create_frame` / `update_frame` / `get_frame` / `list_frames` が `figma_node_id` を受け取り・返却。正規形（`12318:23788`）、ハイフン形（`12318-23788`）、Figma URL のいずれでも指定可能。
- `list_frames` がアーカイブ済み画面を返さないようになりました。
